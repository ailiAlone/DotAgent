- 项目名 DotAgent，是一个 Godot 编辑器 AI 助手插件。游戏场景有：main_menu（主菜单）、game（躲避方块小游戏）、player_stats（玩家属性）、inventory（背包）、settings（设置）。main_menu.gd 的 TitleLabel 文字之前被改成测试文本，已修复回 "DotAgent"。
- 在所有工具测试中，rename_file 会更新脚本中引用的路径。场景 dotagent_test.tscn 绑定了 dotagent_test.gd，重命名后场景中的脚本引用会自动更新。
- run_current_scene 和 stop_running_scene 是安全的一对工具——开得起就关得掉，没有副作用残留。任何时候调试需要真机运行，放心用这两个工具，不要跳过它们。
- 修复了 tool_base.gd 中 _ensure_dir() 的 bug：当文件在 res:// 根目录时，path.rfind("/") 截取出 "res:/"（少一个斜杠）导致报 "Could not create directory: 'res:/'" 错误。增加了根目录检测 + 改用 dir_exists()/make_dir_recursive() 标准变体。
- check_script_syntax 底层 _validate_gdscript 已修复。根因是设 resource_path + reload() 会走 Godot 资源缓存，检测不到语法错误。改为 FileAccess 读取文件内容到 source_code 再 reload()，强制内存编译。2026-06-09 修复。
- 信号连接规则：永远不要在 _ready() 之外连接信号。所有信号连接都在脚本的 _ready() 中通过代码 `.connect()` 完成。永不使用 connect_signal 工具（它写入 [connection] 到 .tscn 不可靠，会被 Godot 覆盖或丢失）。
- 3D skill updated with real-world lessons: (1) Input.is_key_pressed()代替Input.get_vector("ui_*")绕过编辑器快捷键拦截, (2) add_input_action工具有bug需手动execute_gdscript绑定, (3) run_scene_capture无法测试输入, (4) 复制节点不会复制子节点需每个Area3D单独加CollisionShape3D, (5) @onready路径相对于节点自身, (6) CharacterBody3D y坐标公式 = capsule_height/2, (7) Area3D信号.bind(child)参数顺序为signal_args在前bind_args在后, (8) Camera3D.current=true和作为Player子节点

- 理解场景的规则（2026-06-10 用户纠正）：永远不要 read_resource_as_text 读整个 .tscn —— 文件一大就爆 context（coin_collector_3d.tscn 就 9500+ 字符），还得手动解析 sub_resource 引用。正确做法：list_nodes（一行一个节点）→ get_scene_tree → 按需 get_node_properties 看单个节点 → screenshot_editor/screenshot_runtime 看渲染结果。同样写场景不要 execute_gdscript 手写 .tscn，用 create_scene + add_node + set_node_property + remove_node + reparent_node 这套高层 API。
